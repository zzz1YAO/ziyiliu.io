# Homepage v2 改进方案（草案）

## 目标
- 在不改变你“学术主页”定位的前提下，提升专业感、可信度和可维护性。
- 保持当前轻量静态站结构（`index.html` + `styles.css` + `scripts.js`）。

## 我计划做的改进

### 1) 信息一致性与可信度（高优先级）
- 统一“最后更新时间”策略，避免冲突显示。
  - 当前问题：`footer` 里写了固定月份，同时 JS 又自动改成当前年月。
  - 计划方案（默认）：改为“手动维护一个明确日期”，如 `Last updated: 2026-02-17`，不再自动覆盖。
- 统一页面语言设置：
  - 如果主内容为英文，将 `lang` 改为 `en`。
  - 保持文案风格一致（不混用中英文格式日期）。
- 去除或隐藏占位外链（如 `#` 的社媒链接），避免“未完成”观感。

### 2) 内容呈现与学术表达（高优先级）
- 强化 `Publications` 模块的信息密度：
  - 标题、作者、venue/preprint、链接分层更清晰。
  - 保留你“co-first author”重点信息，但视觉层级更自然。
- `Projects` 与 `Publications` 做弱去重：
  - 避免同一成果重复一句话描述，改为“论文侧重学术信息，项目侧重代码/演示/贡献”。
- `About` 段落轻微压缩，首屏更聚焦（身份 + 研究方向 + 代表作）。

### 3) 可访问性与基础 SEO（中优先级）
- 增加基础 SEO 元信息：
  - `meta description`
  - Open Graph（`og:title`, `og:description`, `og:image`）
  - Twitter Card 基本字段
- 提升可访问性：
  - 更具体的头像 `alt` 文本
  - 键盘焦点样式（`focus-visible`）可见
  - 外链安全属性统一（`rel="noopener noreferrer"`）

### 4) 视觉与交互微调（中优先级）
- 在保留你现有简洁风格下升级视觉系统：
  - 提取 CSS 变量（颜色、间距、圆角、阴影）
  - 调整标题和正文的层级对比，让“科研主页”更稳重
- 导航交互改进：
  - 平滑滚动保留
  - 增加当前 section 的高亮（scroll spy，轻量实现）
- 移动端优化：
  - 导航在窄屏下增加更清晰点击区域
  - 减少过长行宽，提升阅读舒适度

### 5) 性能与工程细节（可选）
- 头像图片增加懒加载与解码提示（`loading`, `decoding`）。
- 如果你同意，可减少对整包图标库依赖（后续可改为 SVG）。

## 交付方式（确认后执行）
- 直接修改：
  - `index.html`
  - `styles.css`
  - `scripts.js`
- 改动风格：小步可读、不过度重构，保证你后续易维护。
